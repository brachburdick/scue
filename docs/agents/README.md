# SCUE Agent Workflow Documentation

## Quick Start

Starting a new agent session? Here's the protocol:

1. Determine the agent role (Operator, Architect, or Developer)
2. Start a new conversation (never reuse existing ones)
3. Tell the agent to read the appropriate preamble files from disk:
   - `docs/agents/preambles/COMMON_RULES.md` (always)
   - `docs/agents/preambles/[ROLE]_PREAMBLE.md` (role-specific)
4. Provide the handoff packet (from `handoffs/YYYY-MM-DD/` or generated by the Operator)
5. Wait for the agent to confirm understanding before proceeding

## File Index

| File | Purpose |
|------|---------|
| `preambles/COMMON_RULES.md` | Behavioral contract shared by all roles |
| `preambles/OPERATOR_PREAMBLE.md` | Orchestrator/operator-specific rules |
| `preambles/ARCHITECT_PREAMBLE.md` | Architect-specific rules |
| `preambles/DEVELOPER_PREAMBLE.md` | Implementation agent rules (most detailed) |
| `AGENT_ROSTER.md` | Role definitions, scope boundaries, context injection guide |
| `HANDOFF_CONTRACTS.md` | Format specs for all inter-agent artifacts |
| `ORCHESTRATOR_PROMPT.md` | Full system prompt for Operator sessions |
| `TRANSITION_PLAN.md` | Original transition guide (historical reference) |

## Key Directories

| Directory | Purpose |
|-----------|---------|
| `handoffs/YYYY-MM-DD/` | Handoff packets, organized by date |
| `sessions/YYYY-MM-DD/` | Session summaries, organized by date |
| `specs/` | Feature specs, audit specs, task breakdowns |
| `reviews/` | Code review reports |
| `research/` | Technology investigation findings |

## Archive

`archive/` contains previous versions of workflow documents kept for historical reference:
- `AGENT_PREAMBLE_v1.md` — Original single preamble (replaced by role-specific preambles)
- `AGENT_PREAMBLE_ADDENDUM_v1.md` — First addendum (merged into Developer preamble)

## Updating the Preambles

Preambles are living documents. After each session, the Operator reviews "Preamble Improvement Candidates" from the agent's session summary and updates the appropriate preamble file. This is how the workflow iterates.
