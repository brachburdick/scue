# Handoff Packets

Handoff packets are structured context bundles generated by the Operator or Architect and consumed by implementation agents. Each packet tells one agent exactly what to do, what files to touch, and what to produce.

## Structure

- Organized by date: `handoffs/YYYY-MM-DD/[agent]-[task-slug].md`
- Each packet references on-disk preambles and docs (no embedded copies)
- Packets are consumed once — after the agent session completes, the session summary in `sessions/` becomes the authoritative record of what happened

## Generating Handoffs

- Operator generates handoffs for implementation agents
- Architect generates handoffs when producing task breakdowns for multiple agents
- All handoffs follow the format in `docs/agents/HANDOFF_CONTRACTS.md`

## Preamble Reference

Every handoff includes this at the top, replacing the need to paste full preambles:

```markdown
## Preamble
Read these files before proceeding:
1. `docs/agents/preambles/COMMON_RULES.md`
2. `docs/agents/preambles/[ROLE]_PREAMBLE.md`
```
